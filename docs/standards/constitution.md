# Project Constitution: PRD Web Agent

This project provides a web ui on a specialist Pi agent for creating and refining Product Requirement Documents.  It's being developed to make it easier for non-technical product managers to access the Ensemble workflow.  It is not a full framework, only supports PRD Creating, Refining and Submitting.

> This document defines the non-negotiable standards, guardrails, and quality gates for AI-augmented development.
> Generated by `/init-project` on 2026-02-26. Edit to customize.

---

## 1. Core Principles

### Development Methodology: TDD (Test-Driven Development)

All feature work follows the RED-GREEN-REFACTOR cycle:

1. **RED** — Write a failing test that defines the expected behavior
2. **GREEN** — Write the minimum code to make the test pass
3. **REFACTOR** — Clean up while keeping tests green

No feature is considered complete without accompanying tests written _before_ the implementation.

### Non-Negotiable Rules

1. **No secrets in code** — All credentials via environment variables or secrets manager
2. **Input validation required** — Validate all external input at system boundaries (API routes, user input)
3. **Tests accompany features** — No feature is complete without tests
4. **Agent-only PRD editing** — All PRD content changes happen through agent conversations, never manual text editing
5. **Prisma as single source of truth** — Database schema defined exclusively in `prisma/schema.prisma`

## 2. Tech Stack

### Languages & Frameworks

- **Primary Language:** TypeScript 5.x (strict mode)
- **Framework:** Next.js 14, App Router
- **Runtime:** Node.js 20 (pinned via devbox)

### Data & Services

- **ORM:** Prisma 5 + PostgreSQL 15
- **Auth:** NextAuth.js v5 (OAuth: Google, Okta, Azure AD)
- **Realtime:** Socket.io + Redis adapter
- **Search:** OpenSearch
- **Object Storage:** S3 (LocalStack in dev)
- **Styling:** Tailwind CSS 3

### Testing

- **Unit Testing:** Jest + React Testing Library
- **Integration Testing:** Jest (API route testing)
- **E2E Testing:** Playwright (when applicable)

### Build & Deploy

- **Build Tool:** Next.js built-in (Turbopack in dev)
- **Containerization:** Docker multi-stage build
- **Orchestration:** Helm charts for Kubernetes
- **Dev Environment:** devbox + process-compose

## 4. Approval Requirements

The following changes require explicit user approval before implementation:

### Always Require Approval

- **Architecture changes** — Structural changes to system design, new services, pattern changes
- **New dependencies** — Adding npm packages or system-level dependencies
- **Database schema changes** — Prisma migrations, schema modifications, data model changes

### No Approval Needed

- Reading files or searching code
- Running tests (non-destructive)
- Creating new files in appropriate locations
- Adding tests or documentation
- Development server operations
- jj operations (status, diff, log, describe)
- devbox and process-compose operations

## 5. Agent Delegation Standards

### Specialist Priority

When delegating implementation work:

1. Use framework-specific specialists when available (e.g., `frontend-developer` for React/Next.js, `backend-developer` for API routes)
2. Use `postgresql-specialist` for database work, `infrastructure-developer` for Helm/Docker
3. Use `general-purpose` only for research, not implementation

### Quality Validation

Every implementation must pass through:

1. `test-runner` for coverage validation
2. `code-reviewer` for security and quality
3. `playwright-tester` for E2E (when applicable)

## 6. Project-Specific Conventions

### File Organization

- API routes: `src/app/api/<resource>/route.ts`
- Components: `src/components/<domain>/<ComponentName>.tsx`
- Services: `src/services/<domain>/`
- Tests: Co-located in `__tests__/` directories adjacent to source
- Types: `src/types/`

### Source Control

- **VCS:** jj (Jujutsu) — not raw git
- **Commit style:** `feat(TASK-NNN): description` or `feat: description`
- **Issue tracking:** bd (beads)

### Environment

- `.env.example` is the template; `.env` is local (gitignored)
- devbox pins all system dependencies via Nix
- Docker Compose for local services (postgres, redis, opensearch, localstack)
